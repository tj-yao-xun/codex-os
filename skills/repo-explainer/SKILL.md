---
name: repo-explainer
description: Use this when analyzing an unfamiliar repository and producing a structured explanation of its purpose, architecture, flow, and risks.
---

When using this skill:

1. Start with project purpose:
   - what problem it solves
   - who it is for
   - what kind of repository it is

2. Map the structure:
   - root files
   - main directories
   - entry points
   - config files
   - docs and examples

3. Explain the execution model:
   - how the system starts
   - how control or data flows
   - important dependencies

4. Identify important modules:
   - core logic
   - adapters / integrations
   - tests
   - extension points

5. Evaluate:
   - strengths
   - weaknesses
   - likely assumptions
   - missing pieces

6. Separate:
   - direct evidence
   - inference
   - unknown or unverified points

Output structure:
- Project positioning
- Architecture
- Main flow
- Key files
- Engineering tradeoffs
- Risks / limitations
- Suggested improvements
